`Feature(要素)` API 是一个多部分标准。它提供了创建、修改、查询空间数据的能力。它还为希望 Feature API 的具体实现提供了要求和建议。

它目前有 4 个部分：

- 第一部分，`Core` 部分，是这个 API 标准的强制要求，仅对于空间数据的 **读取** 访问
- 第二部分，`CRS` 部分，扩充了第一部分中几何数据的 **坐标系**，第一部分只支持 WGS84 地理坐标系
- 第三部分，`Filtering` 部分，补充了空间过滤的行为规范，定义了一种类似 SQL 的 **通用查询语言（Common Query Language）**，出于历史原因，英文缩写为 **CQL2**
- 第四部分，`CRUD` 部分，即 `Create/Replace/Update/Delete` - 也就是常说的增删改查四种行为

目前 4 个部分的状态：

| 部分      | 状态 |
| --------- | ---- |
| Core      | 发布 |
| CRS       | 发布 |
| Filtering | 草稿 |
| CRUD      | 草稿 |
